# AEGIS — Personal AI Assistant

AEGIS is an interactive, immersive, highly stylized personal AI assistant designed with a futuristic sci-fi terminal theme (calm, dry wit, highly efficient). It features real-time client-side Speech Recognition (microphone) and Speech Synthesis (text-to-speech voice readout) that coordinate with stunning, responsive CSS-animated rotating SVG rings representing its state.

---

## Key Features

- **Sci-Fi HUD Theme:** Beautiful, dark-mode terminal layout featuring custom typography, a real-time system clock, and a status bar that transitions dynamically based on state.
- **Dynamic State Visualizer:** An interactive SVG core visualizer with rotating outer, middle, and inner rings that change speed, styling, and animations depending on whether the assistant is `IDLE`, `LISTENING`, `PROCESSING` (thinking), or `SPEAKING`.
- **Speech Synthesis (TTS):** Responses generated by AEGIS are dynamically read aloud in a robotic, calm voice using the browser's native text-to-speech API.
- **Speech Recognition (STT):** Dictate your commands directly to AEGIS by tapping the microphone button (available on compatible browsers).
- **Instant Quick Controls:** Fast access to time, date, joke triggers, clear log toggling, and sound mute/unmute buttons.
- **Full Backend Integration:** Powered by **TanStack AI** with seamless provider fallback chain supporting Anthropic Claude, OpenAI, Gemini, and Ollama.
- **Tool Enabled:** Equipped with a weather tool capability (`getWeather`) to retrieve current climate details when asked.

## Setup and Development

### Prerequisites

You must set one or more of the following environment variables to activate the backend intelligence:

```bash
export ANTHROPIC_API_KEY="your-anthropic-key"
export OPENAI_API_KEY="your-openai-key"
export GOOGLE_GENERATIVE_AI_API_KEY="your-gemini-key"
```

If no keys are supplied, the backend falls back to local Ollama (defaulting to the `mistral:7b` model).

### Installation

1. Install dependencies:
   ```bash
   npm install
   ```

2. Start the local development server:
   ```bash
   npm run dev
   ```
   This starts the development server on `http://localhost:3000`.

3. To emulate Netlify environment features and previews:
   ```bash
   netlify dev --port 8889
   ```

## How It Works

1. **State Engine:** The active state is derived reactively from browser status and API loading events. The active class is bound to the top-level app wrapper, driving visual changes.
2. **Text-To-Speech Queue:** On API completion, the new message text is captured and passed to `SpeechSynthesisUtterance`. Voice settings are tuned to a flat, companion-style output (rate `1.02`, pitch `0.9`).
3. **Session Clearing:** Selecting "CLEAR LOG" offsets the displayed slice of messages in React, leaving a pristine clean screen while maintaining session state consistency behind the scenes.
